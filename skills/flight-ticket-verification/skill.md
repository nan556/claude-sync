# 机票行程单核对技能

## 使用场景

核对账单中的机票行程单，核验名字、日期、航班号、出发时间、票号、行程与PDF行程单一致。

## 核心逻辑

```
PDF行程单金额 + 服务费(Col20) + 保险费(Col19) = 应收款(Col22)
```

## 核对流程

```
1. 提取PDF数据：航班号、出发时间、总金额
2. 匹配票号：Excel票号 ↔ PDF文件名中的票号
   - PDF文件名格式：序号-订单号-票号-姓名.pdf
   - 票号在第3部分（索引2）
   - 特殊情况：票号可能分成两部分如 序号-订单号-票号1-票号2-姓名.pdf
3. 判断类型：
   - 改签单 → 只核对名字+票号+金额
   - 正常机票 → 核对名字+票号+航班号+出发时间+行程+金额
4. 金额计算：PDF金额 + 服务费 + 保险费 = 应收款
5. 标记：正确→黄色，错误→红色（修改后标红）
```

## 匹配规则

| 字段 | 改签单 | 正常机票 |
|------|--------|----------|
| 乘客名 | ✓ | ✓ |
| 票号 | ✓ | ✓ |
| 航班号 | ✗ | ✓ |
| 出发时间 | ✗ | ✓ |
| 行程 | ✗ | ✓ |
| 金额 | ✓ | ✓ |

## 票号匹配规则

```python
# PDF文件名格式：序号-订单号-票号-姓名.pdf
# 例如: 11-26118999211031523780-9999467505650-华德玮.pdf

# 票号提取（需处理特殊格式）
parts = filename.replace('.pdf', '').split('-')
if len(parts) == 5:

    passenger = parts[-1]
elif len(parts) == 4:
    ticket_num = parts[2]
    passenger = parts[-1]

# 匹配逻辑：精确匹配 或 长度>=10时的子串匹配
if excel_clean == pdf_ticket:

if len(excel_clean) >= 10 and len(pdf_ticket) >= 10:
    return (excel_clean in pdf_ticket) or (pdf_ticket in excel_clean)
```

## 金额计算

```
应收款 = PDF行程单金额 + 服务费(Col20) + 保险费(Col19)
```

**注意：** 服务费或保险费为0时，使用实际值0，不使用默认值10或5。

## Excel列位置

| 列 | 字段 |
|----|------|
| Col9 | 出发（入住）日（出发时间） |
| Col19 | 保险费 |
| Col20 | 服务费 |
| Col22 | 应收款 |
| Col31 | 发票号码（输出） |

## 标记规则

- **黄色**：应收款单元格标黄，核对正确
- **红色**：应收款单元格标红+红色字体，核对有误

## 航班号提取

PDF文本中的航班号格式：
- CA1333, CZ3911, HU7710（字母+数字）
- 3U8784, 9C8918（数字+字母+数字）
- Y87509（字母+5位数字）

**正则：**
```python
patterns = [
    r'\b([A-Z]{1,2}\d{3,4})\b',  # CA, CZ, HU等
    r'\b(\d[A-Z]\d{4})\b',          # 3U, 9C
    r'\b([A-Z]\d{5})\b',            # Y8系列
]
```

## PDF金额提取

```python
patterns = [
    r'CNY\s*([\d,]+\.?\d*)',  # CNY 1860.00 - 优先匹配
    r'¥\s*([\d,]+\.?\d*)',
```

## 输出

- 核对后Excel文件：源文件路径_机票核对后.xlsx（与源文件同目录）
- 应收款单元格标记颜色（黄色=正确，红色=错误）
- 错误汇总报告

## 错误类型

1. **金额错误**：PDF金额+服务费+保险 ≠ 应收款
2. **航班错误**：Excel航班号 ≠ PDF航班号
3. **时间错误**：Excel出发时间 ≠ PDF时间
