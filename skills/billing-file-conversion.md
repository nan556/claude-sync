# 账单文件转换技能 (Billing File Conversion)

## 功能
将原始账单文件转换为新格式账单，按项目组分小计行，添加黄色背景和SUM公式。

## 输入输出
- **源文件**: `深圳时空数字科技有限公司2026年4月账单(2).xlsx`
- **目标文件**: `深圳时空数字科技有限公司2026年4月账单.xlsx`（格式参考）
- **输出文件**: `深圳时空数字科技有限公司2026年4月账单_转换后.xlsx`

## 表头格式
```
Row 1: 飞鹤航空客户对帐单 (合并 A1:AD1)
Row 2: 客户名称：深圳时空数字科技有限公司 | 对账人：龙晓冰 | 出票日期：2026-4-1--2026-4-30
Row 3: 结算币种: CNY(人民币) | 联系电话：13423938429 | 应收款: =V{合计行}
Row 4: 序号 | 产品类型 | 销售日期 | 成本中心 | 乘客名 | ... | 项目组 | ...
```

## 合并单元格（前3行）
```
A1:AD1  (标题行)
A2:H2   (客户名称)
I2:O2   (对账人)
P2:AD2  (出票日期)
A3:H3   (结算币种)
I3:O3   (联系电话)
P3:X3   (应收款标签)
Y3:AD3  (应收款值 =V{合计行})
```

## 日期格式
- **销售日期（第C列）**: 格式化为 `YYYY/M/D`（如 `2026/4/24`），**去掉时分秒**
- **出发日、到达时间**: 保持原格式

## 项目分组逻辑

### 核心原则
**保持原始文件中数据的相对顺序不变**，只对项目名进行标准化显示。

### 分组规则
1. 原始文件中的数据行按原始顺序排列
2. 被源文件小计行分隔的数据行块视为一个项目组
3. 项目组内可能有不同的原始项目名（如"泰山玻纤展厅"、"泰安纤贼"、"泰安纤"、"泰安玻纤"），这些相似名称应映射到统一的标准名
4. 无项目的数据行放在最后

### 数据读取
```python
def is_subtotal_row(ws, row_idx):
    """检查是否已是小计行（通过 Column 17 的 =SUM 公式判断）"""
    cell = ws.cell(row=row_idx, column=17)
    val = cell.value
    if val and isinstance(val, str) and val.startswith('=SUM'):
        return True
    return False

data_rows = []
group_ids = []  # 记录每行所属的组ID（被同一个小计行分隔的行归为同一组）

current_group = 0
for row_idx in range(5, ws.max_row + 1):
    if is_subtotal_row(ws, row_idx):
        current_group += 1  # 遇到小计行，组ID增加
        continue
    seq = ws.cell(row=row_idx, column=1).value
    if seq is None or str(seq).strip() == '':
        continue
    # 读取数据...
    data_rows.append(row_data)
    group_ids.append(current_group)  # 当前行属于当前组
```

### 写入逻辑
- **保持原始顺序**：按 `group_ids` 顺序写入，不打乱
- **项目列显示**：用标准化项目名替换原始项目名显示
- **小计行**：当组ID变化时插入小计行
- **无项目数据**：放在最后


## 小计公式 (Column Q-V)
每个组结束时插入小计行：
- Q (Col 17): =SUM(Qstart:Qend) 改签费
- R (Col 18): =SUM(Rstart:Rend) 退票费
- S (Col 19): =SUM(Sstart:Send) 保险
- T (Col 20): =SUM(Tstart:Tend) 服务费
- U (Col 21): =SUM(Ustart:Uend) 已收款
- V (Col 22): =SUM(Vstart:Vend) 应收款

## 注意事项
- 源文件已有小计行，读取时必须跳过（通过 Column 17 的 =SUM 公式判断）
- 只读取 Column 1（序号）不为空的数据行
- **总共30列，不要遗漏AB/AC/AD列**
- 项目组是第27列(AA)
- **按原始顺序保持数据相对位置不变**，不打乱顺序
- 合计行V列 = 所有小计行的V列之和
- Y3必须指向合计行的V列
