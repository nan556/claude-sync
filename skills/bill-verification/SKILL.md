# Bill Verification Skill

## Overview

核对账单发票（机票除外）：核验酒店、服务费、保险、高铁票等与PDF发票一致。

**核心逻辑：** 按产品类型汇总金额后匹配发票

## When to Use

- 需要核对账单与PDF电子发票（机票除外）
- 机票核对请使用 flight-ticket-verification 技能

## 产品类型与发票目录映射

| 产品类型 | 发票目录 | 发票金额计算 |
|----------|----------|--------------|
| 国内酒店 | 国内酒店 | 应收款 - 服务费 |
| 火车票 | 高铁票 | 应收款（全额） |
| 国际酒店 | 国际酒店_机票_退费 | 应收款（全额） |
| 国际机票 | 国际酒店_机票_退费 | 应收款（全额，可合并项目组） |
| 退票费 | 国际酒店_机票_退费 | 退票费列(Col18)汇总匹配 |
| 服务费 | 服务费 | 服务费汇总 |
| 保险 | 保险 | 保险费汇总 |

## 匹配规则

### 国内酒店（按项目组汇总匹配）
```
期望PDF金额 = 汇总(应收款 - 服务费)
```
- 按项目组(Col27)聚合所有酒店行的应收款和服务费
- **精确匹配**（不容差）
- 匹配正确：黄色背景
- 未匹配：红色背景（文字保持原色）
- 净值为0或负数的项目（如正负抵消），标记为"异常"，列在人工核对问题中

### 国际酒店（按项目组汇总匹配）
```
期望PDF金额 = 汇总(应收款)
```
- 按项目组(Col27)聚合所有国际酒店行的应收款
- **精确匹配**（不容差）
- 匹配正确：黄色背景
- 未匹配：红色背景（文字保持原色）

### 国际机票（按项目组汇总匹配）
```
期望PDF金额 = 汇总(应收款)
```
- 按项目组(Col27)聚合所有国际机票行的应收款
- 同一项目下多人可合并汇总（如2028+2029项目各2950，合并为5900）
- **精确匹配**（不容差）
- 匹配正确：黄色背景
- 未匹配：红色背景（文字保持原色）

### 退票费（按退票费列汇总匹配）
```
期望PDF金额 = 汇总(退票费)
```
- 退票费列=Col18
- 同一乘客或同一项目的多张退票可合并汇总（如141+141=282）
- **精确匹配**（不容差）
- 匹配正确：黄色背景
- 未匹配：红色背景（文字保持原色）

### 火车票
```
发票金额 = 汇总(应收款)
```

### 服务费/保险
```
发票金额 = 汇总(服务费/保险费)
```

## PDF识别规则

### 识别失败重试机制
PDF识别可能出现异常（如文本提取不完整），当识别结果异常时，需**连续识别3次**，取最优结果：

```python
# 识别失败重试
for attempt in range(3):
    text = extract_pdf_text(pdf_path)
    ticket = extract_ticket_number(text)
    price = extract_price(text)
    if ticket and price > 0:
        break  # 识别成功则停止
```

### 票号匹配规则

### 国内酒店/火车票/服务费/保险
```python
# PDF文件名格式：序号-订单号-票号-姓名.pdf
parts = pdf_filename.replace('.pdf', '').split('-')
invoice_num = parts[2].strip()  # 第3部分是票号

# 金额匹配（精确匹配）
if pdf_amount == expected_amount:
    matched.append(invoice_num)
```

### 国际酒店/国际机票/退票费
```python
# PDF文件名格式：公司名_订单号_日期_金额.pdf
# 例如：深圳时空数字科技有限公司_26952000001890354646_2026-05-09_4879.00.pdf
parts = pdf_filename.replace('.pdf', '').split('_')
invoice_num = parts[1].strip()  # 第2部分是订单号（作为票号）

# 金额匹配（精确匹配）
if pdf_amount == expected_amount:
    matched.append(invoice_num)
```

## 标记规则

- **黄色背景**：应收款单元格标黄，核对正确
- **红色背景**：应收款单元格标红（文字保持原色），核对有误

## 输出

- 核对后Excel文件：xxx_核对后.xlsx（与源文件同目录）
- 应收款单元格标记颜色（黄色=正确，红色=错误）
- 票号写入行末尾空列
- 错误汇总报告

### 票号写入规则（重要）

**核心原则：只处理当前类型，不覆盖其他类型的工作成果**

1. **清除规则**：处理某种类型时，**只清除该类型**的票号，不影响其他类型
   ```python
   # 错误做法：清除所有票号
   for row in range(5, ws.max_row + 1):
       ws.cell(row, 31).value = None  # 会覆盖其他类型的工作！
   
   # 正确做法：只清除当前类型
   for row in range(5, ws.max_row + 1):
       product_type = ws.cell(row, 2).value
       if product_type == '火车票':  # 只处理火车票
           ws.cell(row, 31).value = None
   ```

2. **追加规则**：同一行可能有多种类型的票号（如国内酒店+火车票），用**逗号分隔**
   ```python
   # 如果该行已有票号，追加而非覆盖
   existing = ws.cell(row, 31).value
   if existing:
       ticket_cell.value = f"{existing},{new_ticket}"
   else:
       ticket_cell.value = new_ticket
   ```

3. **颜色标记规则**：只标记当前处理类型的应收款单元格背景色

## 核对报告格式

每次核对完成后，必须生成以下格式的报告：

```
============================================================
发票核对报告
============================================================

【PDF加载与使用情况】
加载PDF总数: X 张
  - 国内酒店: X 张
  - 国际酒店/国际机票/退票费: X 张
  - 高铁票: X 张（如有）
  - 服务费: X 张（如有）
  - 保险: X 张（如有）

使用PDF数: X 张
  - 国内酒店: X 张
  - 国际: X 张

未使用PDF (X张):
  - 订单号1 (金额XXX) - 原因：XXX
  - 订单号2 (金额XXX) - 原因：XXX
  ...

**未使用PDF原因分析：**
- 金额不在该人Excel净额列表中
- 已被其他行先匹配
- Excel中无此人记录
- 识别失败的PDF

【核对结果】
| 类型     | 匹配行数 | 未匹配行数 |
|---------|---------|-----------|
| 国内酒店 | XX行    | X行       |
| 国际酒店 | XX行    | X行       |
| 国际机票 | XX行    | X行       |
| ...     | ...     | ...       |

【人工核对问题】（需人工确认）
| 行号 | 类型   | 项目       | 应收款  | 问题           |
|------|--------|-----------|--------|----------------|
| XXX  | 国内酒店 | 项目名    | XXX    | 未匹配(期望XXX) |
| XXX  | 国内酒店 | 泰山玻纤  | XXX    | 异常(净值0)    |

============================================================
```

**注意：**
- 所有匹配规则均为**精确匹配**，不容差
- 净值为0或负数的项目（如正负抵消）标记为"异常"，列在人工核对问题中
- 未使用PDF需逐个列出订单号和金额，方便人工追溯
